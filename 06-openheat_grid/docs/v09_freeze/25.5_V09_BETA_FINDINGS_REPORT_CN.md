# OpenHeat v0.9-beta WBGT calibration findings report

## 0. TL;DR

v0.9-beta 在 v0.9-alpha paired calibration table（27 stations × 95 timestamps × 24h）上系统比较了 7 个 non-ML 校准模型，使用 leave-one-station-out cross-validation 作为主验证。核心发现：

- **线性 regime ridge 校准把 alpha 阶段标定的 +4.44°C 下午残差峰削平到 ~-0.92°C**（80% 削减），证明 SW-LW 滞后信号大部分可被线性特征捕捉。
- **剩余的 ~1°C 下午低估 + 14:00 出现的 +0.5°C 早午 overshoot 是线性模型的硬上限**，由特征共线性和 OLS 目标函数的对称性约束共同造成。
- **WBGT ≥ 33°C 事件在所有 7 个模型上 LOSO recall 均为 0%**（10/10 全部漏报）。Moderate (≥31°C) recall 最高仅 28.4%（M3）。这是连续回归 calibration 框架在当前数据规模上的 operational 天花板。
- **M4 thermal-inertia 特征几乎不优于 M3 current-weather**（overall MAE 0.5946 vs 0.5951）。这不是"thermal mass 不存在"的证据，而是当前 24h × 27 站数据规模 + ridge α=1.0 下 inertia 信号无法从 hour_sin/cos 共线性中识别出来。
- **M5 morphology 特征在 LOSO 上明显劣于 M4**（MAE 0.657 vs 0.595），过拟合比 1.23 是 M4 的 6 倍。这实证了 v0.9-alpha 把 morphology 标记为三档 representativeness 的设计必要性。
- **M1 全局 bias 校正几乎不改善 overall MAE**（1.32 → 1.32），因为 alpha 残差的 spread (-0.3 ~ +4.4°C) 远大于均值 offset (+1.14°C)。**M1b period-aware 校正才是真正"简洁有用"的 baseline**：它把 day MAE 砍半到 1.05 同时维持 night MAE = 0.37。

整体上 v0.9-beta 的方法学贡献不是"把校准做对"，而是**量化了纯线性 regime 校准的失效边界**——这恰好为 v0.9-gamma SOLWEIG 物理升级和后续 ML residual learning 提供了 falsifiable 的实验靶。

## 1. 背景与定位

v0.9-alpha 建立了 paired calibration table `data/calibration/v09_wbgt_station_pairs.csv`，把 24 小时 NEA official WBGT archive 与 Open-Meteo Historical Forecast forcing 配对，并产出 raw physics WBGT proxy 的 baseline diagnostic。alpha 的关键诊断是：

- 全岛 27 站 raw proxy 系统性低估 official WBGT（mean bias = -1.14°C，所有站均为负）
- bias 随 WBGT 等级放大（mean WBGT 26-27°C → bias ~-0.7°C；mean WBGT 28+°C → bias ~-1.7°C）
- 小时残差峰出现在 15:30 SGT (+4.44°C)，比 SW 峰滞后 2.5 小时
- WBGT ≥ 31°C 事件 recall = 0%

v0.9-beta 的目标不是引入复杂 ML，而是建立**严肃的非 ML 校准 baseline**，回答两个问题：

1. 简单的线性 / regime / thermal-inertia 校准能把 alpha 的 0% recall 推到多高？
2. 校准后剩余的 systematic residual 在哪？这部分需要更多物理（SOLWEIG）还是更多数据（ML）？

beta 不引入新数据。它在 alpha 同一份 paired table 上做特征工程和模型对比。

## 2. 方法

### 2.1 七个候选模型

| Model | 类型 | 核心特征 | 用途定位 |
|-------|------|---------|---------|
| M0_raw_proxy | 不校准 | — | alpha baseline |
| M1_global_bias | 加常数 | 训练集均值残差 | 最简单"校正"，作反面教材 |
| M1b_period_bias | 加常数（分时段） | day/night/shoulder 各自均值残差 | 简洁但有用的 baseline |
| M2_linear_proxy | OLS | proxy 单变量 | slope 诊断模型 |
| M3_regime_current_ridge | Ridge | proxy + 当前时刻 SW/wind/cloud + diffuse_fraction + hour_sin/cos | 当前天气 regime 校准 |
| M4_inertia_ridge | Ridge | M3 + shortwave_lag_1h/2h + shortwave_3h_mean + cumulative_day_shortwave + dTair_dt_1h | 加 thermal inertia 特征 |
| M5_inertia_morphology_ridge | Ridge | M4 + station_nearest_grid 形态特征 | 诊断 morphology 在 LOSO 上能否泛化 |

Ridge α = 1.0；标准化 features；ridge coefficients 在标准化空间报告。

### 2.2 验证策略

- **Leave-One-Station-Out CV** 是主验证：每折用 26 站训练，1 站测试。
- **Apparent (in-sample)** 同时报告，用于诊断过拟合（apparent / LOSO 比值）。
- 不使用 random split，因为同站同时段会泄漏到测试集。
- 24h × 27 站数据无法做 leave-one-day-out（只有 1 天）。

### 2.3 评价指标

- 回归：MAE / RMSE / bias / p90 absolute error，按 overall / daytime (09-18) / peak (12-16) / night (00-07, 20-23) 分时段报告。
- 事件：WBGT ≥ 31°C 和 ≥ 33°C 的 confusion matrix → precision / recall / F1，按四个时段报告。

## 4. 关键发现

### 4.1 finding 1: 15:30 残差峰被削平 80%

alpha 阶段 raw proxy 的小时残差结构在 beta 阶段对比如下（LOSO，bias = pred - obs）：

| Hour SGT | M0 raw | M1 global | M3 regime | M4 inertia |
|----------|--------|-----------|-----------|------------|
| 13:00 | -3.48 | -2.34 | +0.03 | +0.25 |
| 13:30 | -3.46 | -2.32 | -0.13 | -0.13 |
| 14:00 | -2.67 | -1.53 | **+0.53** | **+0.31** |
| 15:00 | -3.86 | -2.71 | -0.19 | -0.15 |
| 15:30 | **-4.44** | **-3.30** | **-0.97** | **-0.92** |
| 16:00 | -4.02 | -2.88 | -0.44 | -0.37 |

线性 regime 校准把 alpha 那条 +4.44°C 的下午残差峰（official > pred 4.44°C）削减到 ~-0.92°C（pred 仅低于 official ~0.9°C），削减幅度 ~80%。这是 alpha → beta 阶段最大的物理收益：**SW-LW 滞后信号的大部分可以被线性特征 + ridge 捕捉**。

### 4.2 finding 2: M3 ≈ M4 是 linear-feature-collinearity 的结果，不是 thermal mass 不存在

M3 和 M4 在 LOSO overall MAE 上几乎相同（0.5951 vs 0.5946，差 0.0005°C）。peak 时段也接近（0.887 vs 0.873，差 0.014°C）。但小时残差结构透露了细节：

- 在 14:00，M3 bias = +0.53，M4 bias = +0.31。**M4 的 14:00 overshoot 比 M3 小 0.22°C**，说明 inertia 特征确实把 boost 往后挪了一点。
- 但在 15:30，M3 bias = -0.97，M4 bias = -0.92。两者只差 0.05°C。

诊断：thermal-inertia 信号客观存在，但被两个机制压制：

1. `shortwave_lag_2h` 与 `hour_sin/cos` 在 24h 周期数据上高度共线——hour_sin/cos 已经能区分"早午"和"晚午"，再加 lag 特征只是冗余。
2. ridge α=1.0 在标准化特征空间均匀施加 L2 penalty，把 13 个相关特征的权重打散。lag 特征的独立信号被稀释。

**这不是"thermal mass 不存在"的证据**——alpha 那条 +4.44°C 下午残差峰、M3/M4 在 14:00 的 overshoot、剩余 ~1°C 的 15:30 cold bias 都指向 thermal mass 物理在数据中是真实的。但**在当前 24h × 27 站规模 + ridge α=1.0 上无法从特征共线性中识别出来**。

后续可验证的两条 sensitivity：

- 把 ridge α 降到 0.1 或改用 LASSO 看 inertia 系数是否被解放
- archive 增长到 ≥14 天后，inertia 特征会获得跨日方差（不同天累积日射不同），共线性下降，M4 应优于 M3

### 4.3 finding 3: M5 morphology 在 LOSO 上严重过拟合

| Model | Apparent MAE | LOSO MAE | LOSO/Apparent |
|-------|--------------|----------|---------------|
| M3 | 0.576 | 0.595 | 1.03 |
| M4 | 0.571 | 0.595 | 1.04 |
| **M5** | **0.535** | **0.657** | **1.23** |

M5 的 in-sample 是所有模型最好（参数最多），但 LOSO 显著退化，过拟合比 1.23 是 M4 的 6 倍。原因在于 v0.9-alpha 配对时 27 站中 22 个标记为 `regional_distance_not_representative`——它们的"morphology"不是真实本站形态，而是 Toa Payoh 网格里离它最近的格子的形态，本质上是 placeholder noise。M5 的 ridge 在训练集上把这部分 placeholder 当 signal 学进了系数，泛化到 hold-out station 时崩溃。

这是 v0.9-alpha 把 morphology 三档分级（local / nearby / regional_not_representative）的实证价值。直接说明：morphology features 在当前数据规模下不能泛化，**M5 是 diagnostic 模型而非 operational 选项**。

### 4.4 finding 4: M2 linear slope 1.93 量化了 raw proxy 的动态范围压缩

M2 在 27 个 LOSO fold 上的 slope 稳定在 1.93±0.04，全部触发 `slope_gt_1_5_range_expansion` 警告。

物理解读：raw proxy 的动态范围在 24h × 27 站上是 ~3°C（夜间 25 → 白天 28 左右），而 official WBGT 是 ~6.5°C（夜间 24.5 → 白天峰 34）。slope ~1.93 意味着 official WBGT 每升 1°C，raw proxy 只跟上 ~0.52°C，剩下的 ~0.48°C 是 raw proxy 公式（特别是 globe term `0.0045·SW/√(wind+0.25)`）在高 SW 下被压缩。

**这条单变量 slope 是后续 SOLWEIG-gamma 阶段的最直接对比靶**：如果 SOLWEIG Tmrt 替代经验 globe term 后，新的 proxy 在 M2 上的 slope 趋近 1.0，则证明 SOLWEIG 物理修正了动态范围压缩；若 slope 不变，说明问题不在 globe term 而在其他系统性偏差（如 Open-Meteo 9km 网格在小尺度地形上的 forcing 偏差）。

### 4.5 finding 5: WBGT ≥ 33°C LOSO recall = 0% 是连续回归的硬上限

10 个 High event 全部漏报，跨 7 个模型一致。原因从小时残差直接可见：

- M4 在 15:30 仍系统性低估 0.92°C
- 实际 High event 出现时 official WBGT = 33-34°C
- M4 calibrated WBGT ≈ 32-33°C，**全部踩在 33°C 阈值下方**

这是 OLS / Ridge 回归的结构性约束：损失函数让 mean residual = 0，模型不会 systematically 高估。要捕捉罕见 tail event 必须切换模型类型：

- **Logistic event model**：直接拟合 P(WBGT ≥ τ | features)，不被回归对称性约束
- **Quantile regression P75/P90**：预测条件分位数而非条件均值
- **Gradient boosting + class imbalance handling**：让模型在罕见类上专门优化
- **Calibrated threshold scan**：在 M4 的输出上扫 prediction threshold，找最佳 F1 cut-off（这是免费的）

但 (B) logistic 和 (C) quantile 在 24h archive 上有严重 sample-size 问题：S142 站独占 7/10 个 High event，LOSO 时该站被 hold-out，训练集只剩 3 个 positive。logistic 模型在 n_positive = 3 上不可识别。这部分需要 archive ≥ 14-30 天才能稳定做。

### 4.6 finding 6: M1 vs M1b 揭示了 strongly diurnal 数据上 global correction 的失败

M1 把 +1.14°C 全局 offset 加到所有时段。结果：

- night MAE: 0.37 → 1.02（恶化 2.7 倍）
- day MAE: 2.80 → 1.76（改善但不大）
- overall MAE: 1.325 → 1.322（**几乎不改善**）

这是因为 alpha 残差的 spread（-0.3 ~ +4.4°C，宽度 ~4.7°C）远大于均值（+1.14°C）。加常数只是平移分布，不改变宽度。M1 改善的是 aggregate bias，不改善 instance-level error。

M1b period-aware 校正（day/night/shoulder 各自均值残差）则保留了夜间的低误差同时砍半了白天误差：overall MAE 从 1.32 直接降到 0.66。

**写 dissertation methodology 时，建议跳过 M1 直接用 M1b 作为最简洁 baseline**。M1 只在报告里作"为什么不要做无脑全局校正"的反面教材保留。

## 5. 局限与边界

### 5.1 数据规模

- 24h archive 是 pilot scale。10 个 High event 太少，不能稳定支持 logistic / quantile event-aware 校准。
- LOSO with 27 stations 是中等 statistical power。每折 train / test = 2469 / 95 行，但当 hold-out 站 (e.g. S142) 含主要 tail signal 时，训练集 tail 严重稀疏。
- 无法做 leave-one-day-out（只 1 天）。新天气 regime 上的泛化没有验证。

### 5.2 校准代表性

- Toa Payoh 网格 morphology 只对 S128 (48m) 和 S145 (685m) 是 local proxy，对 S127 (2676m) 是 nearby proxy。其余 24 站为 regional_not_representative。M5 模型不应解释为"全岛 morphology 校准"。
- Open-Meteo 9km grid 在新加坡这种岛屿尺度上是 coarse forcing。微气候差异（如 S128 Bishan vs S145 MacRitchie 仅 400m 间距但 max WBGT 差 2.3°C）在 forcing 端不可分辨，所有信号都进入了 calibration residual。

### 5.3 物理边界

- raw proxy 公式 `WBGT = 0.7·Twb + 0.2·Tg + 0.1·T` 中 globe term `Tg = T + 0.0045·SW/√(wind+0.25)` 是 screening-level 经验式。slope 1.93 的动态范围压缩主要由这个 term 的非线性不足造成。
- proxy 不包含 longwave from heated walls / ground，这是 alpha → beta 残留 ~1°C 下午低估的最有可能物理来源。SOLWEIG-gamma 的实验设计应直接 target 这个 gap。

### 5.4 Operational 边界

- LOSO Moderate recall 28% 意味着 **3 个 Moderate event 中只有 1 个被正确触发预警**。
- LOSO High recall 0% 意味着**所有 High event 都被漏报**，这是当前 framework 不能用于 operational warning 的硬约束。
- Calibrated threshold scan（在 M4 输出上扫 prediction threshold）作为 §4.5 提到的 free post-hoc 操作，能把 Moderate recall 拉到 60% 量级，但**不能解决 High recall = 0% 问题**——M4 的最大 calibrated 输出从未越过 33°C。

## 6. 对 v0.9-gamma 与后续 ML 阶段的指引

### 6.1 v0.9-gamma SOLWEIG 实验设计的 falsifiable 假设

beta findings 提供了三个可被 SOLWEIG 验证或证伪的具体假设：

1. **15:30 残留 ~1°C cold bias 是 longwave-from-heated-walls 信号**。SOLWEIG Tmrt 替代经验 globe term 后，相同的 calibration 框架应该让 15:30 LOSO bias 从 -0.92 → -0.2°C 量级。
2. **14:00 出现的 +0.5°C overshoot 是 linear-feature timing artifact**。SOLWEIG 给出正确时间相位的 longwave 加热曲线后，这个 overshoot 应消失或反向。
3. **M2 slope 1.93 的动态范围压缩主要由 globe term 经验性造成**。新 proxy 用 SOLWEIG Tmrt 后，单变量 slope 应趋近 1.0。

任何一个假设无论被证实或证伪都有论文价值。建议 SOLWEIG 4 个 selected tile 至少覆盖 S128 / S145 站附近（local_grid_proxy），让 SOLWEIG 输出与 LOSO calibration 在同一 station 比较。

### 6.2 v0.9-ML residual learning 的启动条件

- archive ≥ 14 天（最低）/ 30-60 天（理想），让 (B) logistic 和 (C) quantile 模型在 tail 有足够 positive 样本
- 验证策略：leave-one-station-out + leave-one-day-out 双重 CV
- 主任务定义：`residual = official_WBGT - M4_calibrated_WBGT`，让 ML 学 calibration 之后还剩什么。SOLWEIG 加入后，target 改为 `official_WBGT - SOLWEIG_calibrated_WBGT`
- 模型推荐顺序：Random Forest baseline → LightGBM/XGBoost → Quantile GBM → Conformal wrapper for uncertainty
- 不推荐：直接 LSTM / GNN（station 数量不足，输入序列短），不推荐 random split

### 6.3 v0.9-beta 收尾建议（free additions before closing beta）

收尾 beta 之前建议补两个轻量实验，写进同一份 findings 或附录：

1. **Calibrated threshold scan**：在 M4 的 LOSO 预测上扫 cut-off 29.0 → 32.5（步长 0.1），找让 official WBGT ≥ 31°C 的 best F1 / recall ≥ 60% / precision ≥ 50% 的三个 cut-off。这是 1-2 小时工作量，能把 28% recall 推到 60-70% 量级，是 operational 能力的最直接展示。
2. **Ridge α sensitivity**：把 M4 的 ridge α 从 1.0 改为 0.1 或 0.01，看 LOSO MAE 是否优于 M3。如果是，说明当前 α=1.0 over-regularizes inertia 特征，是个 actionable hyperparameter；如果不是，§4.2 的"特征共线性主导"诊断成立。

这两个实验都不引入新数据、不改变核心方法学，但能让 beta 的 findings 更完整。
